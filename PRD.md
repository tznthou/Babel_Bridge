# 📗 PRD (產品需求文件)
# Babel Bridge - 打破語言與聽覺障礙的即時字幕系統

> 本文件以 **行為導向 (BDD) + 使用者需求導向** 為核心,定義 Babel Bridge 的產品目標、使用者故事與功能需求。

---

## 🧭 一、專案簡介 (Overview)

### 核心問題
全球有超過 4.66 億聾啞人士與聽力受損者,他們在觀看網路影片時面臨巨大障礙。雖然部分平台 (如 YouTube) 提供字幕功能,但:
- 自動生成字幕準確度不足 (約 70-80%)
- 許多影片完全沒有字幕
- 小型平台與個人網站通常不支援字幕
- 外語影片缺乏翻譯字幕

此外,外語學習者、在公共場合需要靜音觀看的人、以及多語言內容創作者也需要更好的字幕解決方案。

### 解決方案
Babel Bridge 是一個 Chrome 擴充功能,使用 AI 語音辨識技術,將**任何**網路影片的音訊即時轉換成準確的字幕,並支援多語言翻譯。就像為巴別塔搭建一座橋樑,消除語言與聽覺的障礙。

### 產品價值
- **無障礙**: 讓聾啞人士能獨立享受網路影片內容
- **多語言**: 突破語言限制,支援全球 90+ 語言
- **通用性**: 支援所有網站,不受平台限制
- **即時性**: 延遲僅 4-5 秒,接近即時體驗
- **免費**: 完全開源,使用者僅需自備 API Key

## 👥 二、目標使用者 (Target Users)

| 使用者類型 | 需求或痛點 | 預期收益 |
|------------|------------|----------|
| **聾啞人士 / 聽力受損者** | 大部分影片缺乏字幕或字幕品質不佳,無法理解內容 | 獲得高準確度即時字幕,獨立享受影片內容 |
| **外語學習者** | 想同時看到原文與翻譯,加強語言學習 | 雙層字幕顯示,提升學習效率 |

| **多語言內容創作者** | 需要快速生成多語言字幕 | 自動生成準確字幕,節省時間成本 |
| **教育工作者** | 需要為學生提供無障礙學習材料 | 任何影片都能加上字幕,提升教學品質 |

## 🧩 四、使用者故事 (User Stories)

> 採用 BDD (Given-When-Then) 格式

### 🎬 Epic 1: 音訊擷取與辨識

#### Story 1.1: 啟用字幕功能
**As a** 使用者  
**I want to** 在觀看影片時一鍵啟用字幕  
**So that** 我能快速開始使用字幕功能

**Scenario: 成功啟用字幕**
- **Given** 我已安裝 Babel Bridge Extension
- **And** 我正在觀看一個 YouTube 影片
- **When** 我點擊 Extension 圖示並按下「啟用字幕」按鈕
- **Then** 系統應該請求音訊擷取權限
- **And** 在我授權後開始擷取音訊
- **And** 在 popup 顯示「字幕已啟用」狀態

**Scenario: 未授權音訊擷取**
- **Given** 我點擊「啟用字幕」按鈕
- **When** 瀏覽器彈出權限請求
- **And** 我點擊「拒絕」
- **Then** 系統應該顯示友善的錯誤訊息
- **And** 提示我需要授權才能使用功能

#### Story 1.2: 即時語音辨識
**As a** 使用者  
**I want to** 影片音訊能被即時轉換成文字  
**So that** 我能理解影片內容

**Scenario: 辨識英文內容**
- **Given** 字幕功能已啟用
- **And** 我正在觀看英文影片
- **When** 影片播放超過 3 秒
- **Then** 系統應該開始辨識音訊
- **And** 在 4-5 秒內顯示第一段字幕
- **And** 字幕內容準確反映影片音訊

**Scenario: 辨識中文內容**
- **Given** 字幕功能已啟用
- **And** 我正在觀看中文影片
- **When** 影片持續播放
- **Then** 系統應該正確辨識中文語音
- **And** 字幕包含正確的繁體或簡體中文字

#### Story 1.3: 處理影片控制
**As a** 使用者  
**I want to** 字幕能跟隨影片播放狀態  
**So that** 字幕顯示與影片同步

**Scenario: 暫停影片**
- **Given** 字幕正在顯示
- **And** 影片正在播放
- **When** 我暫停影片
- **Then** 字幕應該停止更新
- **And** 當前字幕保持顯示

**Scenario: 快轉影片**
- **Given** 影片正在播放
- **When** 我拖曳進度條到新位置
- **Then** 系統應該重新開始音訊擷取
- **And** 從新位置開始產生字幕

### 🎨 Epic 2: 字幕顯示與自訂

#### Story 2.1: 顯示字幕
**As a** 使用者  
**I want to** 字幕清晰地顯示在影片上  
**So that** 我能舒適地閱讀字幕

**Scenario: 基本字幕顯示**
- **Given** 語音辨識已產生文字
- **When** 字幕資料傳送到 Content Script
- **Then** 字幕應該疊加在影片下方中央
- **And** 字幕有半透明黑色背景提升可讀性
- **And** 使用白色文字,字體大小適中

**Scenario: 長字幕自動換行**
- **Given** 辨識產生的文字超過一行寬度
- **When** 字幕顯示時
- **Then** 文字應該自動換行
- **And** 最多顯示 2-3 行
- **And** 不會遮擋影片重要內容

#### Story 2.2: 自訂字幕樣式
**As a** 使用者  
**I want to** 能調整字幕外觀  
**So that** 字幕符合我的閱讀偏好

**Scenario: 調整字體大小**
- **Given** 我開啟 Popup 設定面板
- **When** 我移動「字體大小」滑桿
- **Then** 字幕字體應該即時改變大小
- **And** 新設定應該被儲存

**Scenario: 改變字幕位置**
- **Given** 我在設定中選擇「字幕位置」
- **When** 我選擇「上方」或「下方」
- **Then** 字幕應該移動到指定位置
- **And** 不會遮擋影片播放器控制列

**Scenario: 調整透明度**
- **Given** 我想讓字幕背景更透明
- **When** 我調整「背景透明度」
- **Then** 字幕背景的透明度應該改變
- **And** 確保文字仍然清晰可讀

#### Story 2.3: 字幕歷史記錄
**As a** 使用者  
**I want to** 能查看之前的字幕  
**So that** 我能回顧錯過的內容

**Scenario: 開啟字幕歷史**
- **Given** 字幕已經顯示一段時間
- **When** 我點擊「顯示歷史」按鈕
- **Then** 應該展開一個側邊欄
- **And** 顯示最近 50 條字幕記錄
- **And** 包含時間戳記

### 🌍 Epic 3: 多語言翻譯

#### Story 3.1: 啟用翻譯
**As a** 外語學習者  
**I want to** 將字幕翻譯成我的母語  
**So that** 我能同時學習外語和理解內容

**Scenario: 選擇目標語言**
- **Given** 我已啟用字幕功能
- **When** 我在 Popup 中點擊「翻譯」開關
- **And** 選擇目標語言為「繁體中文」
- **Then** 系統應該開始翻譯字幕
- **And** 在原文下方顯示翻譯結果

**Scenario: 雙層字幕顯示**
- **Given** 翻譯功能已啟用
- **When** 新的字幕產生
- **Then** 應該先顯示原文字幕 (4-5 秒延遲)
- **And** 然後在下方顯示翻譯字幕 (6-8 秒延遲)
- **And** 兩層字幕樣式可區分 (不同字體大小或顏色)

#### Story 3.2: 翻譯品質
**As a** 使用者  
**I want to** 翻譯結果準確且自然  
**So that** 我能正確理解影片內容

**Scenario: 語境相關翻譯**
- **Given** 原文是「Bank」(銀行或河岸)
- **When** 影片內容是關於金融
- **Then** 翻譯應該正確識別為「銀行」
- **And** 而非「河岸」

**Scenario: 保留專業術語**
- **Given** 原文包含技術術語 (如 "API", "JavaScript")
- **When** 翻譯字幕時
- **Then** 專業術語應該保持原文
- **And** 不進行錯誤的音譯

#### Story 3.3: 翻譯語言切換
**As a** 多語言使用者  
**I want to** 能快速切換翻譯語言  
**So that** 我能靈活使用不同語言

**Scenario: 切換目標語言**
- **Given** 當前翻譯語言是英文
- **When** 我在 Popup 選擇日文
- **Then** 系統應該立即切換到日文翻譯
- **And** 清除舊的英文翻譯字幕
- **And** 開始顯示日文翻譯

### ⚙️ Epic 4: 設定與管理

#### Story 4.1: API Key 設定
**As a** 新使用者  
**I want to** 能輕鬆設定 OpenAI API Key  
**So that** 我能開始使用服務

**Scenario: 首次安裝引導**
- **Given** 我剛安裝 Babel Bridge Extension
- **When** 我首次開啟 Popup
- **Then** 系統應該顯示設定嚮導
- **And** 清楚說明需要 OpenAI API Key
- **And** 提供「取得 API Key」的連結(指向 OpenAI Platform)
- **And** 說明預估成本($0.37/小時)

**Scenario: 輸入並驗證 API Key**
- **Given** 我在設定畫面中
- **When** 我將 API Key 貼入輸入欄位
- **And** 點擊「儲存並驗證」按鈕
- **Then** 系統應該呼叫 OpenAI API 測試 Key 是否有效
- **And** 如果有效,顯示「✓ API Key 驗證成功」
- **And** 如果無效,顯示「✗ API Key 無效,請檢查後重試」
- **And** API Key 應該安全儲存在 chrome.storage.local

**Scenario: API Key 格式錯誤**
- **Given** 我在輸入 API Key
- **When** 我輸入的格式不符合 OpenAI Key 格式(不以 sk- 開頭)
- **Then** 系統應該即時顯示格式錯誤提示
- **And** 不允許儲存直到格式正確

**Scenario: 修改已存在的 API Key**
- **Given** 我已經設定過 API Key
- **When** 我開啟設定頁面
- **Then** 應該顯示「API Key 已設定」狀態
- **And** API Key 應該部分遮罩顯示(如 sk-...xyz)
- **And** 提供「更新 API Key」按鈕
- **When** 我點擊「更新 API Key」
- **Then** 允許我輸入新的 API Key
- **And** 重新驗證後更新

- **And** 在 Popup 中顯示警告訊息

#### Story 4.3: API Key 安全性
**As a** 使用者  
**I want to** 我的 API Key 被安全保護  
**So that** 我不用擔心資料外洩

**Scenario: 本地儲存**

- **And** 不會傳送到任何第三方伺服器(除了 OpenAI)

**Scenario: 遮罩顯示**
- **Given** 我開啟設定頁面查看 API Key
- **Then** API Key 應該部分遮罩顯示
- **And** 提供「顯示完整 Key」的眼睛圖示
- **When** 我點擊眼睛圖示
- **Then** 短暫顯示完整 Key(3秒後自動隱藏)

## ⚙️ 五、功能需求 (Functional Requirements)

| 編號 | 功能名稱 | 說明 | 優先級 | 驗收準則 |
|------|----------|------|--------|----------|
| **FR-1** | 音訊擷取 | 使用 chrome.tabCapture 擷取 tab 音訊流 | P0 | 能成功擷取音訊且無噪音 |
| **FR-2** | 音訊切段 | Rolling Window 每 3 秒切一段,重疊 1 秒 | P0 | 音訊段時長準確,重疊區正確 |
| **FR-3** | 音訊編碼 | Web Worker 編碼音訊為 mp3 格式 | P0 | 編碼不阻塞 UI,檔案大小合理 |
| **FR-4** | 語音辨識 | 呼叫 Whisper API 轉錄音訊 | P0 | 辨識準確度 ≥ 90%,回傳 timestamp |
| **FR-5** | 斷句優化 | 比對重疊區,合併破碎句子 | P0 | 句子完整,無不當斷句 |
| **FR-6** | 字幕注入 | Content Script 在影片上疊加字幕 | P0 | 字幕正確顯示,不影響影片播放 |
| **FR-7** | 字幕樣式 | 支援字體大小、顏色、位置、透明度調整 | P1 | 樣式設定即時生效並儲存 |
| **FR-8** | 影片同步 | 監聽 play/pause/seek 事件並同步字幕 | P0 | 字幕與影片狀態完全同步 |
| **FR-9** | 翻譯功能 | 使用 GPT-4o-mini 翻譯字幕 | P1 | 翻譯準確且自然,延遲 < 8 秒 |
| **FR-10** | 雙層字幕 | 同時顯示原文與翻譯 | P1 | 兩層字幕可區分,不互相遮擋 |
| **FR-11** | 語言選擇 | 支援 20+ 目標語言選擇 | P1 | 切換語言立即生效 |
| **FR-12** | API Key 輸入 | 提供 UI 供使用者輸入 OpenAI API Key | P0 | 輸入欄位清晰,有格式驗證 |
| **FR-13** | API Key 驗證 | 驗證 API Key 是否有效 | P0 | 能正確判斷 Key 有效性,顯示結果 |
| **FR-14** | API Key 儲存 | 安全儲存 API Key 到 chrome.storage.local | P0 | Key 加密儲存,僅本地存取 |
| **FR-15** | 首次設定引導 | 引導新用戶設定 API Key | P0 | 流程清晰,提供取得 Key 的連結 |

## 🧩 九、風險與假設 (Risks & Assumptions)

### 技術風險

| 風險 | 影響 | 機率 | 緩解措施 |
|------|------|------|----------|
| Whisper API 響應時間過長 | 字幕延遲超過 5 秒,影響體驗 | 中 | • 優化音訊編碼<br/>• 減少音訊段大小<br/>• 使用 CDN 加速 |
| 部分網站無法擷取音訊 | 功能在某些網站不可用 | 高 | • 明確列出支援網站清單<br/>• 研究 DRM 繞過方案(需注意法律)<br/>• 提供替代方案 |
| API 成本過高 | 使用者負擔太重,影響採用率 | 中 | • 提供成本估算工具<br/>• 優化 API 使用(快取、批次處理)<br/>• 探索替代 API |
| 記憶體洩漏 | 長時間使用導致瀏覽器變慢 | 中 | • 定期清理音訊緩衝<br/>• 限制字幕歷史數量<br/>• 進行壓力測試 |

### 商業風險

| 風險 | 影響 | 機率 | 緩解措施 |
|------|------|------|----------|
| OpenAI 調整 API 政策或定價 | 成本大幅上升或功能不可用 | 低 | • 設計模組化架構,便於切換 API<br/>• 追蹤 OpenAI 政策更新<br/>• 準備備用方案 (Google, Azure) |
| 競品出現 | 市場份額被搶佔 | 中 | • 專注核心用戶(聾啞社群)<br/>• 持續優化準確度與速度<br/>• 建立社群與品牌 |
| Chrome Web Store 審核未通過 | 延遲發布 | 低 | • 仔細遵循政策<br/>• 準備詳細說明文件<br/>• 預留審核時間 |

### 使用者體驗風險

| 風險 | 影響 | 機率 | 緩解措施 |
|------|------|------|----------|
| 首次使用門檻高 | 使用者流失 | 中 | • 提供詳細設定嚮導<br/>• 製作教學影片<br/>• 優化 onboarding 流程 |
| 字幕遮擋重要內容 | 影響觀看體驗 | 中 | • 提供位置調整選項<br/>• 智慧偵測影片內容區域<br/>• 允許拖曳字幕 |
| 誤判或錯誤辨識 | 使用者信任度降低 | 中 | • 提供手動修正功能<br/>• 收集錯誤回饋<br/>• 持續優化辨識模型 |

### 假設 (Assumptions)

1. **使用者願意自備 OpenAI API Key**: 假設目標使用者能接受此門檻
2. **網路連線穩定**: 假設使用者有穩定的網路連線供 API 呼叫
3. **Chrome 瀏覽器佔主流**: 優先支援 Chrome,後續再擴展到其他瀏覽器
4. **影片音訊品質良好**: 假設大部分影片有清晰的音訊
5. **OpenAI API 可靠性**: 假設 API 有 99%+ 可用性
